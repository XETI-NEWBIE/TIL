# 📊 토스증권 댓글 수집 및 LLM 증강 시스템 (AI 프로젝트)

## 1. 프로젝트 개요
본 프로젝트는 **Django 기반 웹 애플리케이션**에서 **Selenium**을 활용하여 토스증권의 동적 커뮤니티 데이터를 실시간으로 수집하고, **생성형 AI(LLM)**를 연동하여 데이터 정제 및 증강 파이프라인을 구축하는 것을 목표로 합니다[cite: 1, 42].

---

## 2. 기술 스택
*   **Framework:** Django (Python)[cite: 1, 42]
*   **Automation:** Selenium WebDriver, webdriver-manager[cite: 1, 42]
*   **Data Analysis:** Pandas, re (정규표현식)[cite: 1, 10, 42]
*   **AI/LLM:** LangChain, OpenAI API (GMS)[cite: 1, 10, 42]
*   **Frontend:** HTML5, CSS3, JavaScript (SSE Streaming)[cite: 1, 42]

## 4. 핵심 학습 내용

### 💡 실시간 스트리밍 UI (SSE 기술)
*   서버에서 작업이 끝날 때까지 기다리지 않고, 진행 상황(로그)을 사용자에게 즉시 전달하기 위해 Django의 `StreamingHttpResponse`와 JavaScript의 `EventSource`를 결합한 SSE(Server-Sent Events) 기술을 익혔습니다
*   데이터 처리 단계별로 `yield`를 사용하여 실시간으로 로그를 전송하고, 최종 완료 시 `JSON` 데이터를 파싱하여 화면을 동적으로 렌더링하는 방법을 학습했습니다

### 💡 동적 크롤링 및 웹 자동화
*   SPA(Single Page Application) 구조인 토스증권에서 원하는 요소를 안정적으로 찾기 위해 `WebDriverWait`를 활용한 명시적 대기 기법을 적용했습니다[cite: 6, 60, 75].
*   `webdriver-manager`를 도입하여 드라이버 경로 설정 문제를 자동화하고, `Headless` 옵션과 `User-Agent` 설정을 통해 서버 환경에서의 봇 탐지 우회 및 자원 효율성을 높였습니다

### 💡 LLM 파이프라인 설계 및 연동
*   단순 검색을 넘어 LLM을 파이프라인 중간 단계에 배치하여 데이터의 '품질'을 높이는 경험을 했습니다[cite: 9, 64].
*   프롬프트 엔지니어링을 통해 AI로부터 원하는 형식(JSON 배열 등)의 응답을 받아내고, 이를 파이썬의 `ast.literal_eval` 등을 통해 코드 상에서 즉시 활용 가능한 데이터로 변환하는 과정을 거쳤습니다

## 5. 프로젝트 소감
처음에는 Selenium의 느린 속도와 복잡한 비동기 페이지 로딩 때문에 데이터를 가져오는 것조차 쉽지 않았습니다. 특히 드라이버 경로가 꼬이거나 LLM 호출 중 에러가 발생해 서버가 중단될 때 많은 어려움을 겪었으나, **webdriver-manager**와 **try-except 기반의 예외 처리**를 통해 안정성을 확보할 수 있었습니다

무엇보다 단순히 "데이터를 뽑아내는 것"에서 끝나는 것이 아니라, AI 모델을 이용해 데이터를 정제하고 증강하여 사용자에게 가치 있는 통합 UI로 보여주는 과정을 통해 **전체적인 데이터 서비스의 생명 주기**를 완벽히 이해할 수 있었던 뜻깊은 시간이었습니다
