

# 01-PJT : 파이썬을 활용한 API 데이터 수집 (Weather Data)


## 1. 프로젝트 개요
OpenWeatherMap API를 사용하여 서울의 실시간 날씨 데이터를 가져오고, 이를 내가 원하는 형태로 가공하는 과정을 수행했음. 단순히 데이터를 가져오는 것을 넘어, 복잡한 JSON 구조를 파악하고 필요한 정보만 쏙쏙 골라내어 정리하는 능력을 길렀음.

---


## 2. 사용된 핵심 문법 및 개념 정리 (상세)
이번 프로젝트를 진행하며 코드에 적용한 파이썬의 핵심 기능들을 정리함.

### 1) **f-string (문자열 포맷팅)**
- **코드 예시**: `f"https://api...?lat={lat}"`
- **설명**: 문자열 앞에 f를 붙이고 중괄호 {} 안에 변수 이름을 넣으면, 변수의 값이 문자열 안에 자동으로 들어감. 주소(URL) 중간에 위도, 경도, API 키를 끼워 넣을 때 아주 유용하게 사용했음.

### 2) **딕셔너리(Dictionary)와 리스트(List) 구조**
- **딕셔너리 `{key: value}`**: 단어 뜻풀이처럼 '이름표(Key)'와 '값(Value)'이 짝지어 있는 형태. (예: `{'temp': 24.5}`)
- **리스트 `[value1, value2]`**: 데이터가 순서대로 줄 서 있는 형태. 인덱스(번호)로 꺼내야 함. (예: weather 데이터는 `[ ]`로 감싸져 있어서 리스트임.)
- **구조 파악**: API 데이터는 딕셔너리 안에 리스트가 있고, 그 안에 또 딕셔너리가 있는 **복잡한 상자 구조**였음.

### 3) **JSON 데이터 변환**
- **코드 예시**: `requests.get(url).json()`
- **설명**: 인터넷상에서 데이터를 주고받을 때 쓰는 표준 형식인 JSON을 파이썬이 이해할 수 있는 딕셔너리/리스트 형태로 바꿔주는 기능임.

### 4) **데이터 접근 방식 (.get vs 대괄호)**
- **`.get('key')`**: 해당 키가 없으면 에러 대신 None을 돌려줌. **안전하게 데이터를 꺼낼 때** 사용했음.
- **`['key']`**: 해당 키가 없으면 에러(KeyError)가 발생함. **확실하게 있는 데이터**(예: 앞에서 확인한 weather)를 꺼낼 때 사용했음.

### 5) **round 함수 (반올림)**
- **코드 예시**: `round(값, 2)`
- **설명**: 소수점이 너무 길게 나올 때 사용하는 함수. 두 번째 숫자가 2면 '소수점 셋째 자리에서 반올림해서 **둘째 자리까지 보여줘**'라는 뜻임.

## 3. 단계별 구현 내용
문제 1번부터 4번까지 단계적으로 기능을 확장하며 구현했음.

### A. 데이터 수집 및 키값 확인 (`problem1.py`)
- `requests` 라이브러리를 통해 서버에 데이터를 요청하고 받아왔음.
- `.keys()` 함수를 써서 데이터 꾸러미(딕셔너리)에 어떤 이름표(Key)들이 있는지 목록만 쫙 뽑아봄.
- 데이터의 전체적인 지도를 그리는 단계였음.

### B. 필요한 데이터만 추출 (`problem2.py`)
- 전체 데이터 중에서 날씨 수치 정보인 `main`과 날씨 상태 정보인 `weather`만 딱 골라냄.
- 새로운 딕셔너리를 만들어서 내가 필요한 정보만 담는 **'요약본'**을 만들었음.

### C. 데이터 한글화 및 심화 접근 (`problem3.py`)
- 영어로 된 Key(예: temp, humidity)를 한글(온도, 습도)로 바꿨음.
- **어려웠던 점**: `weather` 데이터가 딕셔너리인 줄 알았는데, 알고 보니 **리스트(`[ ]`)**였음.
- **해결**: `weather[0]`처럼 대괄호와 숫자 0을 써서 **'첫 번째 데이터'**를 꺼낸 뒤에 내부 정보에 접근했음. 이 구조를 이해하는 게 가장 중요했음.

### D. 데이터 가공 및 최종 구조화 (`problem4.py`)
- **섭씨 온도 계산**: API가 주는 온도는 켈빈(K) 단위라 우리가 쓰는 섭씨로 바꾸기 위해 `- 273.15` 계산을 수행함.
- **데이터 구조화**: 최종 결과를 '기본' 정보와 '날씨' 정보 두 그룹으로 나누어 정리함.
- 코드를 깔끔하게 하기 위해 `main_data` 같은 변수를 따로 만들어서 긴 경로를 줄여 썼음.

## 4. 트러블 슈팅 (에러 해결 과정)
코드를 짜면서 마주친 에러들과 해결 방법임.

### 1) **모듈 호출 에러** (`TypeError: 'module' object is not callable`)
- **상황**: `import pprint`라고만 쓰고 `pprint(result)`를 실행하니 에러가 남.
- **원인**: 도구함(모듈) 자체를 실행하려고 해서 생긴 문제.
- **해결**: `from pprint import pprint`라고 써서 도구함 속의 **'pprint 기능'을 콕 집어 가져오거나**, `pprint.pprint()`로 경로를 다 써주는 방식으로 해결했음.

### 2) **키 에러** (`KeyError`)
- **상황**: `result['weather']['description']`이라고 썼는데 키를 못 찾는다고 에러가 남.
- **원인**: `weather`는 **리스트(`[ ]`)**라서 이름표(Key)로 바로 찾을 수 없었음.
- **해결**: 데이터 생김새를 print로 다시 확인하고, `result['weather'][0]['description']`으로 **순서 번호(index)**를 추가해서 해결했음.

## 5. 느낀 점

- **데이터의 생김새 확인이 first**: 무작정 코드를 치기 전에 `print()`나 `pprint()`로 데이터가 딕셔너리인지 리스트인지 눈으로 보고 시작하는 게 시간을 아끼는 길임을 깨달았음.
- **변수명의 중요성**: `temp_c`(섭씨), `main_data` 처럼 변수 이름을 누가 봐도 알기 쉽게 지어놓으니, 나중에 코드가 길어져도 헷갈리지 않았음.

    에러를 읽고 고치는 과정에서 파이썬 문법을 더 깊게 이해하게 됨.
